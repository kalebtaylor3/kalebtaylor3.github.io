Project

This project is a systems-focused UE5 C++ procedural dungeon generator built to showcase production-ready tools and clean presentation. The generator builds a logical grid plan first (deterministic, seedable), then assembles a dungeon around designer-placed anchors so level artists can steer the layout. A target-biased grid walk creates the main path, stochastic branches add variety, and an adjacency pass derives wall/opening masks so rooms connect cleanly. A "Cinematic Mode" stages a debug reveal (tiles, path line) before a final flash that spawns the 3D rooms. The system is theme-ready: rooms can be authored as archetype Blueprints (start/main/branch/wave/end) and plugged in-even pre-dressed-so a full dungeon can be assembled from designer-made pieces.

My Work

I designed and implemented the entire system in C++: grid math, seed control, anchor-aware pathing (target-biased random walk + BFS connectors), branch generation, and per-tile data. I wrote the neighbor-derived opening logic (per-side collision gating) and a robust two-pass spawn that configures walls after all rooms exist, preventing flicker and ensuring no "boxed-in" starts-even with placed anchors. I added editor-tunable debug styling (outline/fill/line) and a Cinematic Mode pipeline to capture the reveal cleanly. Gameplay hooks include room triggers, ambient and wave enemy spawns with Niagara VFX, safe player spawn on the start tile, and Navmesh/AI-ready layouts. The architecture exposes archetype tags and decoration hooks, enabling designers to drop in art-ready modular rooms so the generator can assemble a fully realized dungeon from strategic pieces.
