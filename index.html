What I Do

Systems Design

I design technical systems that are structured, testable, and easy to extend. A lot of that experience comes from game programming work in Unity and Unreal, where I built combat logic, AI systems, procedural generation, and progression features that had to stay readable and stable as projects grew.

C# Application Logic

C# is the language I use most often for production work. I use it to build gameplay systems, tools, save and progression features, AI logic, UI flows, and data-driven behaviours, and it is one of the main reasons my game programming background translates well into general software roles.

C++ and Technical Problem Solving

I use C++ when a project needs tighter control over architecture, performance, and lower-level systems work. That includes custom combat frameworks, procedural generation, animation-driven systems, AI behaviours, and data-heavy logic. I am comfortable debugging edge cases, tracing execution flow, and building features that need to stay readable as they grow.
