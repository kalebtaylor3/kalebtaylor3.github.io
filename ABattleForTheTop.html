Project

A Battle for the Top is an ongoing team project where I own the programming work across multiple gameplay and progression systems. From a software perspective, it is a broad systems integration project that combines character movement, combat, deck management, UI, persistence, and AI behaviour into a single maintainable codebase.

Technical Focus

The project revolves around connecting several interdependent systems without letting the code become brittle. It includes player state, card-driven abilities, encounter logic, progression, save-related behaviours, and zone-specific mechanics, all of which need clear ownership and stable interfaces to scale cleanly.

My Work

As the sole programmer, I designed and implemented the project's core technical systems in C#. That includes movement, combat, card-driven actions, boss behaviour, progression, UI flows, and save-related features. A big part of the work has been keeping those systems modular so new cards, hazards, and zone mechanics can be added without turning the project into one large dependency chain. It is one of my strongest examples of owning a full application layer from architecture through implementation and iteration.
